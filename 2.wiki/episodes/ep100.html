Claude Mythos, Fable 5, 그리고 다음 국면은?

Fable 5·Mythos 공개를 기점으로 경쟁축이 사전학습에서 포스트 트레이닝·데이터셋·루프 엔지니어링으로 옮겨가고, RSI와 주권 AI가 다음 국면을 연다.

핵심 포인트

  • 스케일이 커질수록 사전학습보다 포스트 트레이닝의 중요성이 커진다.
  • 프론티어 모델의 실력은 데이터셋 엔지니어링에서 갈린다.
  • 루프 엔지니어링이 새로운 언어로 등장하고 테스트 타임 컴퓨트가 상업적 변수다.
  • 다리오 아모데이의 글을 따라 RSI 타임라인과 AI의 전략 자산화를 짚는다.
  • 컴퓨팅 스케일 경쟁이 주권 AI로 번지며 한국의 독자 파운데이션 모델 기회가 논의된다.
  • Claude Code 이후 엔지니어링의 일하는 방식 자체가 바뀐다.

마무리

Fable 5 바이브 체크, Claude Code 이후 엔지니어링과 100회 이후 계획

본문 엑기스

AI 프론티어 100회는 지난 3년의 변화를 돌아보는 자리에서 출발한다. 두 진행자는 그 변화를 Anthropic의 Fable 5와 Mythos급 모델 공개라는 사건으로 압축하고, 거기서 다음 국면을 향한 질문들을 끌어낸다 — 이제 모델 경쟁은 어디에서 갈리는가.

첫 결론은 경쟁축의 이동이다. 스케일이 충분히 커진 지금, 추가적인 사전학습보다 포스트 트레이닝이 모델의 성격과 능력을 결정한다. 같은 베이스 모델이라도 어떤 후처리를 거치느냐가 사용감과 안전성, 도구 사용 능력을 좌우한다는 것이다.

그 후처리의 핵심 역량으로 데이터셋 엔지니어링이 지목된다. 프론티어의 실력은 더 이상 '데이터를 많이 모았다'가 아니라, 어떤 데이터를 어떻게 생성하고 정제하고 검증하느냐의 문제다. 데이터셋 엔지니어링이 곧 모델 엔지니어링이 된다.

능력은 도메인을 넘는다. AI가 생물학 같은 영역으로 확장되는 방식이 논의되고, 그만큼 스타트업의 꿈과 자본이 만나는 속도가 빨라진다. 프론티어 네트워크 안에서 정보가 빠르게 공유되는 것 자체가 경쟁력이 된다는 점도 짚는다.

모델의 이름도 그냥 이름이 아니다. Mythos와 Fable이라는 명명은 능력이 곧 위험이 되는 지점, 즉 보안 질문을 던진다. 강한 모델일수록 무엇을 할 수 있느냐와 무엇을 막아야 하느냐가 같은 무게로 따라온다.

여기서 다리오 아모데이의 글이 등장한다. 두 사람은 그의 글을 따라 RSI(재귀적 자기개선)의 타임라인을 읽는다. 모델이 다음 모델의 개발을 돕는 흐름이 현실의 일정표 위에 올라오면서, AI가 단순한 제품이 아니라 전략 자산이 되고 시장도 그렇게 반응한다.

이 국면을 설명하는 새 언어가 '루프 엔지니어링'이다. 모델을 한 번 호출해 답을 받는 방식이 아니라, 루프 안에서 반복 호출하고 검증하고 고쳐 가며 결과를 빚어내는 설계를 가리킨다. 일을 시키는 단위가 '한 번의 응답'에서 '하나의 루프'로 바뀐다.

루프 엔지니어링은 곧 비용 구조의 문제이기도 하다. Fable 5의 가격과 테스트 타임 컴퓨트 — 추론 시점에 연산을 더 써서 품질을 사는 방식 — 그리고 Fable API의 비용이 어디까지 상업적으로 합리적인지가 루프 설계의 경제성을 정한다.

RSI는 추상이 아니라 Anthropic의 자기개선 궤도라는 구체적 맥락에서 다뤄진다. 모델이 도구를 만들고 그 도구가 다음 모델을 더 빨리 만들게 하는 선순환이 어디까지 왔는지를 가늠한다.

그리고 이 모든 것은 컴퓨팅 스케일 경쟁으로 수렴한다. 충분한 연산을 누가 쥐느냐가 곧 주권 AI 경쟁이 된다. AI가 전략 자산이 된 세계에서 컴퓨팅은 국가·진영 단위의 문제로 올라선다.

그 틈에서 한국의 독자 파운데이션 모델 기회가 논의된다. 거대 컴퓨팅 경쟁을 정면으로 좇기 어렵더라도, 데이터셋 엔지니어링과 포스트 트레이닝 역량으로 비집고 들어갈 공간이 있다는 것이다.

마무리는 다시 현장으로 돌아온다. Fable 5의 바이브 체크와 인간적 성찰, Claude Code 이후 엔지니어링이 일하는 방식 자체를 어떻게 바꾸는지, 그리고 AI 프론티어 커뮤니티의 역할과 100회 이후의 계획으로 끝맺는다.
