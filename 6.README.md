# AI Frontier Wiki

`aifrontier.kr`(한국어 AI 팟캐스트, 진행: 노정석·최승준·김성현) 에피소드를 수집·추출해 사람이 읽기 좋은 위키로 만드는 작업 repo다. `local-coding-model-watch`의 운영 규율(결정적 로컬 파이프라인 + 네트워크/LLM 명시적 opt-in + 상태머신 기록 + 기본 비활성 스케줄러)을 그대로 상속한다.

## Scope

- 언어: 한국어 `/ko`만.
- 원문 깊이: 사이트 에피소드 페이지 + `llms.txt` 텍스트만 (YouTube 자막 제외).
- 작업 주체: 추출(엑기스)은 MacBook **Hermes 에이전트 + DeepSeek Pro**(OpenAI 호환, `deepseek` 어댑터). 수집·렌더는 LLM 없이 결정적으로 수행.

## Layout

```text
0.prompt/        Hermes/DeepSeek 추출 프롬프트(정기 재사용)
1.specs/         설계서·HTML IA·검증 체크리스트·launchd 템플릿·fixtures
2.wiki/          ★ 사람이 읽는 위키 (index.html + episodes/epNN.html + keywords/index.html)
3.sources.md     원천 지도(sitemap·llms.txt·RSS·robots 정책)
6.README.md      이 문서
assets/          wiki.css + wiki.js (로컬 자산, CDN 의존 없음)
data/            episodes.json, extracts/*.json, keyword-index.json + 스키마
state/           run-status / current-state / change-history.jsonl / scheduler-state
logs/            회차별 + 스케줄러 로그
scripts/         aifrontier_wiki.py(결정적 CLI) + run_aifrontier_wiki_cycle.sh
config/          local.env.example (실제 키는 커밋 금지)
```

대용량 원문(원본 HTML/텍스트)은 운영 규칙대로 `~/BaseCamp/06_Artifacts/aifrontier-wiki/raw/` 에 저장한다. 정제된 위키·상태·인덱스만 `02_Knowledge`에 둔다.

## Conservative CLI

기본 실행은 네트워크/LLM 없이 동작하는 결정적 파이프라인이다. 라이브 수집과 LLM 추출은 명시적 opt-in이다. CLI는 모델을 다운로드하거나 추론을 임의로 돌리지 않으며, 시크릿 파일을 만들지 않는다.

```bash
python3 scripts/aifrontier_wiki.py validate          # files + schemas + invariants + data/state/render 정합성
python3 scripts/aifrontier_wiki.py build-keyword-index
python3 scripts/aifrontier_wiki.py render-wiki        # canonical 2.wiki/ = data/extracts/의 순수 함수
python3 scripts/aifrontier_wiki.py selfcheck          # 렌더러 완전성 회귀 가드(temp 렌더)
python3 scripts/aifrontier_wiki.py render-wiki --demo-fixture 1.specs/fixtures/ep96-extract.fixture.json  # preview/에만 출력
python3 scripts/aifrontier_wiki.py fetch-index --allow-network               # Phase 1: sitemap -> episodes.json (+ llms.txt snapshot)
python3 scripts/aifrontier_wiki.py fetch-pages --allow-network --limit 5     # Phase 1: originals -> 06_Artifacts
python3 scripts/aifrontier_wiki.py fetch-pages --allow-network --refresh     # incremental re-check of fetched episodes
python3 scripts/aifrontier_wiki.py extract --allow-llm --adapter deepseek --limit 5  # Phase 2 (not yet implemented)
```

Phase 1 수집 노트:

- `fetch-pages`는 에피소드별 `page.html` · `text.txt` · `next-data.json` · `meta.json`을 `~/BaseCamp/06_Artifacts/aifrontier-wiki/raw/epNN/`(repo 밖)에 저장한다.
- 변경 감지는 **원본 바이트가 아니라 추출 텍스트의 해시**로 한다 (사이트가 매 렌더마다 nonce/build-id를 넣어 raw 해시는 매번 바뀌기 때문). 덕분에 주간 회차에서 실제 변경된 에피소드만 재추출한다.
- 예의: User-Agent 명시, 요청 간 지연(`AIFRONTIER_FETCH_DELAY_SECONDS`, 기본 2초), 1회 재시도, 응답 바이트 상한. `--limit`로 회당 처리량 제한.

핵심 불변식: **canonical `2.wiki/`는 canonical `data/extracts/`의 순수 함수**다. `render-wiki`는 관리 대상 페이지(`index.html`, `keywords/index.html`, `episodes/ep*.html`)를 먼저 비우고 재생성하므로 삭제·변경된 에피소드의 유령 페이지가 남지 않는다. `--demo-fixture`는 `preview/`(gitignore)로만 렌더하며 canonical 출력·데이터·상태를 건드리지 않는다.

Outputs (canonical render):

- `data/keyword-index.json` (extracts 재집계)
- `2.wiki/index.html`, `2.wiki/episodes/epNN.html`, `2.wiki/keywords/index.html`
- `state/run-status.json`, `state/current-state.json` (카운트 동기화), `state/change-history.jsonl`

`validate`는 스키마(타입·enum·id 패턴), 교차 불변식(ep_id/엔티티 id 유일성, 키워드 label/type 안정성), 그리고 데이터·상태·렌더 페이지 정합성(고아 페이지 금지, 카운트 일치, keyword-index 최신 여부)을 모두 강제한다.

라이브 수집은 `--allow-network` 가 있어야만 동작하고 robots/sitemap/llms.txt를 진입점으로 사용한다. LLM 추출은 `--allow-llm --adapter deepseek` + `config/local.env`의 엔드포인트/키가 있어야만 동작한다.

스케줄러 래퍼는 `state/scheduler-state.json`으로 기본 비활성이며 7일 주기, 14일 가드를 둔다. `1.specs/launchd/aifrontier-wiki.plist.template`은 템플릿일 뿐 명시적 승인 없이 로드하지 않는다.

## Phases

- **Phase 0 — scaffold**: 폴더/스펙/스키마/자산/CLI 스켈레톤. 네트워크 0. ✅
- **Phase 1 — collect**: `fetch-index` → `fetch-pages`로 ko 전 에피소드 원문 아카이브. ✅ (ep45, ep75–100 등 27편 수집)
- **Phase 2 — extract**: DeepSeek Pro 배치 추출 → `data/extracts/*.json`. (다음)
- **Phase 3 — render**: `render-wiki`로 인덱스·에피소드·키워드 페이지 생성.
- **Phase 4 — schedule**: 표본 검수 후 주간 스케줄러 활성화(신규/변경 에피소드만 증분).
