# Source Manifest — aifrontier.kr

## Purpose

AI Frontier 팟캐스트의 한국어 에피소드를 수집해 엑기스 위키로 컴파일하기 위한 원천 지도.

## Location

- Site root: `https://aifrontier.kr/`
- Korean episodes: `https://aifrontier.kr/ko/episodes/ep{N}`
- Sitemap: `https://aifrontier.kr/sitemap.xml`
- AI-agent content index: `https://aifrontier.kr/llms.txt` (llmstxt.org 포맷)
- RSS: 한국어/영어 피드 제공 (사이트 기재)
- YouTube: `https://www.youtube.com/@chester_roh` (현재 범위 밖)

## Access policy

- `robots.txt` 확인 결과 `User-agent: *` / `Allow: /` — 전체 크롤 허용. 그래도 무차별 크롤링이 아니라 sitemap·llms.txt 인덱스 기반으로만 접근한다.
- 예의 바른 수집: 요청 간 지연, `max_results`/`limit` 상한, 1회 재시도, User-Agent 명시.
- 개인 연구·지식 정리 목적의 로컬 보관만 한다. 원문을 외부에 재배포하지 않는다.
- ToS/robots가 바뀌면 즉시 수집 정책을 재검토한다.

## Collection scope (locked)

- 언어: `/ko` 한국어만.
- 깊이: 에피소드 페이지 본문/쇼노트 + `llms.txt` 텍스트. YouTube 자막 제외.
- 대상 식별: sitemap.xml의 `/ko/episodes/ep{N}` URL 집합. 약 24개(ep45, ep75–100 사이 일부 결번). 정확한 목록은 `fetch-index`가 채운다.

## What agents may do

- `fetch-index --allow-network`: sitemap.xml + llms.txt를 파싱해 `data/episodes.json`을 갱신(신규/변경 diff).
- `fetch-pages --allow-network`: ko 에피소드 원문을 `06_Artifacts/aifrontier-wiki/raw/epNN/`에 저장(page.html, text, meta.json, content_hash).
- `extract --allow-llm --adapter deepseek`: 원문 → 엑기스 JSON(`data/extracts/epNN.json`).
- `render-wiki` / `build-keyword-index`: 추출 결과로 위키와 키워드 인덱스를 결정적으로 생성.

## What agents must not do

- 네트워크/LLM 플래그 없이 외부 호출하지 않는다.
- 시크릿(DeepSeek 키/엔드포인트)을 커밋하지 않는다.
- 원문 전체를 본진 Obsidian으로 옮기지 않는다. 선별된 엑기스/브리프만 승격 검토.
- 대용량 원문을 `02_Knowledge`에 두지 않는다(→ `06_Artifacts`).
